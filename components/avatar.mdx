---
title: Avatar
description: A client component that renders the user's Discord avatar sourced from the Zustand discord-user store, with a skeleton fallback while data loads.
---

The `Avatar` component is a simple client component located at `components/avatar.tsx`. It reads the `LanternUserResponse` data from the `useDiscordUserStore` Zustand store and renders the user's Discord avatar as a circular image.

## Behaviour

- **Before data loads** — renders a 48 × 48 px pulsing skeleton (`animate-pulse`) with `bg-secondary` fill and a `border-primary` ring.
- **After data loads** — renders a `next/image` element sourced from `data.metadata.display_avatar_url`, sized at 48 × 48 px (`size-12`) with a `border-primary` ring and `object-cover`.

Because the store is populated by `DiscordStatus` (which fetches from lantern.rest), `Avatar` never makes its own network request.

## Props

The component accepts no props.

## Source

```tsx
'use client';

import { useDiscordUserStore } from '@/stores/discord-user';
import Image from 'next/image';

export default function Avatar() {
  const data = useDiscordUserStore(state => state.data);

  if (!data) {
    return (
      <div className='size-12 animate-pulse rounded-full border-2 border-primary bg-secondary' />
    );
  }

  return (
    <Image
      src={data?.metadata.display_avatar_url}
      alt=''
      width={100}
      height={100}
      className='size-12 rounded-full border-2 border-primary object-cover'
    />
  );
}
```

## Usage

```tsx
import Avatar from '@/components/avatar';

<Avatar />
```

The component is used in two places in the codebase:

- `app/page.tsx` — in the top header alongside the name and `DiscordStatus`.
- `app/blogs/layout.tsx` — in the back-link header at the top of every blog post.

<Note>
  `Avatar` depends on `DiscordStatus` being mounted somewhere in the same React tree (or higher) so that the store is populated. If `DiscordStatus` is absent, the skeleton state is shown indefinitely.
</Note>
